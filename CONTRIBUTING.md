If you do not agree to those terms, do not submit a contribution.

This applies to **all contributions**, including trivial changes.

## Contribution workflow

1. Open an issue first for large changes (to align on approach).
2. Keep PRs focused and small where possible.
3. Add tests for behavioral changes.
4. Update docs when you change behavior or APIs.
5. Ensure your PR includes:
   - a clear description
   - rationale / tradeoffs
   - any breaking changes called out
